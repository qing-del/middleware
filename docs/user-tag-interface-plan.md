## 1. 背景与目标

标签用于描述笔记和主题的附加语义，是用户内容分类和检索的重要维度。本阶段需要补齐 user 端标签接口，让用户可以创建、查看、删除自己的标签，并将标签绑定或解绑到笔记和主题上。

## 2. 范围与边界

### 2.1 本次范围

- 开发 user 端标签接口（Controller + Service + Mapper + DTO/VO）。
- 支持标签查询、创建、软删除。
- 支持将标签绑定到笔记或主题，也支持解除绑定。

## 4. 接口设计

统一前缀：`/api/tags`

### 4.1 查询标签列表

- 方法与路径：`GET /api/tags`
- 处理逻辑：
	1. 查询当前用户创建的所有标签。
	2. 仅返回当前用户自己的标签列表。

### 4.2 创建标签

- 方法与路径：`POST /api/tags`
- 请求体示例：

```json
{
	"tag_name": "技术"
}
```

- 处理逻辑：
	1. 标签所有者自动设为当前登录用户。
	2. 写入标签基础信息。
	3. 默认按系统约定初始化软删除状态。

### 4.3 删除标签

- 方法与路径：`DELETE /api/tags/{id}`
- 处理逻辑：
	1. 校验标签归属权。
	2. 执行软删除，将 `is_deleted` 置为 `1`。
	3. 保留关联历史，避免直接物理删除。

### 4.4 绑定标签

- 方法与路径：`POST /api/tags/assign`
- 请求体示例：

```json
{
	"tagId": 123,
	"targetId": 456,
	"targetType": "note"
}
```

- 处理逻辑：
	1. 校验标签是否属于当前用户。
	2. 校验目标资源类型是否为 `note` 或 `topic`。
	3. 校验目标资源是否属于当前用户。
	4. 建立绑定关系。

### 4.5 解除绑定

- 方法与路径：`POST /api/tags/remove`
- 请求体建议：与绑定接口保持一致。
- 处理逻辑：
	1. 校验标签与目标资源的归属关系。
	2. 解除标签与笔记或主题之间的绑定。

## 5. 开发草案细化

### 5.1 请求字段建议

- `tag_name`：必填。
- `tagId`：必填。
- `targetId`：必填。
- `targetType`：必填，取值建议为 `note` 或 `topic`。

### 5.2 返回结构建议

- 查询列表：返回标签列表或分页结果。
- 创建、删除、绑定、解除绑定：返回统一成功状态即可。

### 5.3 访问控制

- 所有标签操作都必须以当前登录用户为边界。
- 绑定和解除绑定时必须先确认目标资源属于当前用户。
